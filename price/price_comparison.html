⑤ 关键结论

  • input_tokens ≠ 发出去的总量：它只是「按全价算的部分」。真实发出量 = input + cache_write + cache_read。
  • 三种输入 token 单价差极大：cache_read(0.1×) ≪ input(1×) < cache_write(1.25×)。多轮循环命中 cache_read 是省钱核心。
  • 最强对最强：DeepSeek-R 比 Opus 4.8 便宜约一个数量级（输入/输出 ~9–11×，读缓存 ~3.6×）。
  • 便宜 ≠ 划算：要结合任务难度、成功率、循环轮数综合看——贵模型少走几轮、少返工，总成本可能反而低。这是第 18 章「成本控制 / 模型路由」的主题。
